# iBizSimAssistant 项目规则

## Git 分支管理规则

### 核心原则
- **所有代码更改必须在 `develop` 分支上进行**
- **严禁在 `main` 分支上进行任何操作**
- 用户负责从 `develop` 合并到 `main`

### 分支说明
- **main**: 生产分支，仅包含稳定版本代码
- **develop**: 开发分支，所有开发工作在此进行

### 工作流程
1. 始终在 `develop` 分支上进行开发
2. 所有代码更改、修复、新功能都在 `develop` 分支提交
3. 测试验证在 `develop` 分支完成
4. 用户手动将 `develop` 合并到 `main`

### 禁止操作
- ❌ 在 `main` 分支上进行任何代码修改
- ❌ 在 `main` 分支上提交更改
- ❌ 在 `main` 分支上合并其他分支
- ❌ 未经用户批准切换到 `main` 分支

### 强制检查
在执行任何 Git 操作前，必须确认当前分支为 `develop`：
```bash
git branch  # 确认当前分支标记为 *
```

### 项目配置文件
- 配置文件: `.trae/rules/project_rules.md`
- 此文件定义项目的核心开发规则
- 任何规则变更需经用户批准
